# دليل الإعداد والاستخدام السريع

## المتطلبات
- Laravel 11+
- PHP 8.1+
- MySQL/MariaDB
- Bootstrap 5.3+
- Font Awesome 6.4+

## خطوات التثبيت

### 1. تشغيل الـ Migrations
```bash
php artisan migrate --force
```

تم إنشاء الجداول التالية تلقائياً:
- `chat_polls` - استبيانات الشات
- `chat_poll_votes` - أصوات الاستبيانات
- `employees` - بيانات الموظفين
- `attendance_records` - سجل الحضور
- `kpi_metrics` - مؤشرات الأداء
- تحديث `broadcasts` بأعمدة targeting
- تحديث `social_cases` بأعمدة المراحل

### 2. مسح الـ Cache
```bash
php artisan cache:clear
php artisan config:cache
```

### 3. السماح بـ Routes
تأكد من أن routes الجديدة مسجلة:
```bash
php artisan route:list | grep -E "chat-polls|hr\.|activity"
```

### Feature ١: إصلاح مشكلة الكاش

**الموقع:** `broadcasts.index`

**الخطوات:**
1. اذهب إلى "رسائل عاجلة"
2. أرسل رسالة جديدة
3. اسمح للمستخدمين برفضها
4. انقر على زر "إعادة تفعيل"
5. الرسالة ستظهر مجددًا لجميع من رفضوها

### Feature ٢+٣: الإشعارات والصوت

**الموقع:** `notifications` + `layouts.modern` (JavaScript)

**الخطوات:**
1. ستتلقى إشعارات تلقائياً كل ١٥ ثانية

### Feature ٥: استبيانات الشات

**الموقع:** `chat.index` (المحادثة الجماعية)

**الخطوات:**

5. النتائج تُحدّث فوراً دون الحاجة لـ refresh

**لإغلاق الاستبيان:** POST `/api/chat-polls/{poll}/close`

### Feature ٩: نظام الموارد البشرية

**الموقع:** Sidebar → "الموارد البشرية" (لـ مدير/محاسب فقط)

#### أ) لوحة تحكم HR (`hr.dashboard`)
- عدد الموظفين النشطين
- إجمالي الرواتب
- عدد الحاضرين اليوم
- روابط سريعة للعمليات

#### ب) إدارة الموظفين (`hr.employees.*`)

**إضافة موظف:**
1. اذهب إلى "الموظفون"
2. انقر "+ إضافة موظف جديد"
3. اختر المستخدم من قائمة المستخدمين غير المرتبطين
4. أدخل:
   - القسم (مثل "مبيعات")
   - المسمى الوظيفي
   - تاريخ التعيين
   - الراتب الشهري
5. انقر حفظ

**تعديل موظف:**
1. ابحث عن الموظف في القائمة
2. انقر على أيقونة التعديل ✎
3. عدّل البيانات المطلوبة
4. يمكن تغيير الحالة: نشط / غير نشط / في إجازة
5. انقر تحديث

#### ج) تسجيل الحضور (`hr.attendance.*`)

**تصفية حسب الشهر والسنة:**
1. اذهب إلى "الحضور والغياب"
2. اختر الشهر والسنة
3. انقر "تصفية"

**إضافة سجل حضور:**
1. في أسفل الصفحة، اختر:
   - الموظف
   - التاريخ
   - وقت الحضور (اختياري)
   - وقت الانصراف (اختياري)
   - الحالة: حاضر / غائب / متأخر / إجازة
2. انقر "إضافة السجل"

#### د) مؤشرات الأداء (`hr.kpi.*`)

**إضافة KPI:**
1. اذهب إلى "مؤشرات الأداء"
2. في قسم الإضافة:
   - اختر الموظف
   - أدخل اسم المؤشر (مثل "إتمام المشاريع")
   - الهدف (مثل 100)
   - التحقيق الفعلي (مثل 85)
   - فترة التقييم (من - إلى)
3. يُحسب تلقائياً: 85/100 = 85%

**رموز المستوى:**
- 🟢 ممتاز: ≥ 90%
- 🔵 جيد: ≥ 75%
- 🟡 مقبول: ≥ 60%
- 🔴 ضعيف: < 60%

**الفائدة:** مرونة في جمع البيانات مع ضمان الجودة في المرحلة الثانية

## استكشاف الأخطاء

### مشكلة: الإشعارات لا تصل
**الحل:**
1. تحقق من `php artisan queue:work` (إن كان configured)
2. تحقق من تفعيل JavaScript بالمتصفح
3. مسح browser cache: Ctrl+Shift+Delete

### مشكلة: صوت الإشعارات لا يعمل
**الحل:**
1. تحقق من إعدادات الصوت بالمتصفح
2. ابدأ بـ user interaction أولاً (انقر على الصفحة)
3. تحقق من Web Audio API: `console.log(window.AudioContext || window.webkitAudioContext)`

### مشكلة: استبيانات الشات لا تظهر
**الحل:**
1. تأكد من تشغيل الـ migration: `php artisan migrate:status`
2. تحقق من الـ routes: `php artisan route:list | grep chat-polls`
3. مسح cache: `php artisan cache:clear`

### مشكلة: سجل النشاط فارغ
**الحل:**
1. تحقق من تفعيل `ActivityLogService`:
   ```bash
   grep -r "ActivityLogService" app/
   ```
2. أنشئ مصروف/حالة جديدة لاختبار
3. تحقق من الـ logs: `tail -f storage/logs/laravel.log`

### مشكلة: HR routes لا تعمل
**الحل:**
1. تحقق من الـ role: `auth()->user()->roles`
2. تأكد من أن المستخدم له role "مدير" أو "محاسب"
3. اختبر:
   ```bash
   php artisan tinker
   >>> auth()->loginUsingId(1)
   >>> auth()->user()->roles
   ```
