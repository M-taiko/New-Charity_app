### 2. إضافة Database Locks للعمليات المالية
**المشكلة:** Race conditions في العمليات المتزامنة
**الملفات:** `TreasuryService.php`, `CustodyTransferService.php`
**الحل:**
```php
$treasury = Treasury::where('id', $treasuryId)->lockForUpdate()->first();
$custody = Custody::where('id', $custodyId)->lockForUpdate()->first();
```

**الأولوية:** عالية جداً ⭐⭐⭐⭐⭐
**التأثير:** يمنع التلاعب بالأرصدة

---

### 3. تصحيح حساب الرصيد المتبقي
**المشكلة:** `getRemainingBalance()` لا يستخدم `fresh()`
**الملف:** `app/Models/Custody.php`
**الحل:**
```php
public function getRemainingBalance()
{
    return $this->amount - $this->spent - $this->returned - $this->pending_return;
}

// يجب أن يصبح:
public function getRemainingBalance()
{
    // استخدام DB query مباشر لضمان الدقة
    $fresh = static::find($this->id);
    if (!$fresh) return 0;

    return $fresh->amount - $fresh->spent - $fresh->returned - $fresh->pending_return;
}
```

**الأولوية:** عالية ⭐⭐⭐⭐
**التأثير:** دقة الأرصدة في كل مكان
